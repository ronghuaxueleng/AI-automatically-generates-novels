# Novel.ai风格功能使用指南

本项目已添加类似Novel.ai的高级功能，包括Lorebook世界观管理系统和增强的上下文记忆系统。这些功能可以帮助您创建更加连贯、一致的小说内容。

## Lorebook世界观管理系统

Lorebook是一个强大的世界观管理工具，可以帮助您组织和管理小说中的角色、地点、事件等元素。

### 主要功能

1. **条目管理**：创建、编辑和删除Lorebook条目
2. **分类系统**：将条目分类为角色、地点、事件、物品等
3. **关键词触发**：为每个条目设置关键词，当生成内容时自动触发相关设定
4. **优先级设置**：为条目设置优先级，控制在上下文空间有限时的插入顺序
5. **条目搜索和过滤**：快速查找和筛选条目

### 使用方法

1. 点击界面右侧的"Lorebook"按钮打开Lorebook面板
2. 点击"新建条目"创建新的Lorebook条目
3. 填写条目信息，包括：
   - 名称：条目的名称
   - 分类：条目的类型（角色、地点、事件等）
   - 关键词：触发条目的关键词，多个关键词用逗号分隔
   - 内容：条目的详细描述
   - 优先级：条目的重要程度（1-10，数字越大优先级越高）
4. 点击"保存"按钮保存条目
5. 在生成内容时，系统会自动检测关键词并将相关条目的内容添加到提示词中

### 提示和技巧

- 为重要角色创建详细的Lorebook条目，包括外貌、性格、背景故事等
- 使用多个关键词增加触发几率，例如角色的全名、简称、别名等
- 为重要地点创建Lorebook条目，描述其地理位置、历史、文化等
- 优先级设置对于控制上下文空间的使用非常重要，请合理设置

## 增强的上下文记忆系统

记忆管理器可以帮助AI记住长篇小说中的情节、角色和事件，确保生成内容的连贯性和一致性。

### 主要功能

1. **章节记忆**：自动记录和管理已生成的章节内容
2. **角色记忆**：跟踪角色在不同章节中的出场和发展
3. **事件记忆**：提取和记录故事中的关键事件
4. **记忆压缩**：智能压缩旧章节内容，节省上下文空间
5. **记忆摘要**：自动生成故事摘要，帮助AI理解整体情节

### 使用方法

1. 点击顶部菜单栏的"记忆管理"按钮打开记忆管理器设置面板
2. 配置记忆管理器设置：
   - 记忆的最大章节数：系统记忆的最大章节数量
   - 启用记忆压缩：是否压缩旧章节内容
   - 压缩比例：压缩的程度（0.1-0.9）
   - 优先记忆最近内容：是否优先保留最近的内容
   - 自动总结长文本：是否自动总结长文本
   - 总结阈值：触发自动总结的字符数
3. 点击"保存设置"按钮保存设置
4. 系统会自动在生成新章节时更新记忆，并在提示词中添加相关记忆信息

### 记忆管理器的工作原理

1. **记忆收集**：系统会自动从已生成的章节中收集信息
2. **记忆处理**：对收集的信息进行处理，提取关键内容
3. **记忆压缩**：对旧章节内容进行智能压缩，保留关键信息
4. **记忆注入**：在生成新内容时，将相关记忆注入到提示词中
5. **记忆更新**：在新内容生成后，更新记忆库

### 提示和技巧

- 对于长篇小说，建议启用记忆压缩功能
- 可以通过"导出记忆"功能备份记忆数据
- 如果发现AI忘记了某些重要情节，可以调高记忆的最大章节数
- 记忆管理器会自动从人物设定中提取角色信息，请确保人物设定格式正确

## 结合使用两个系统

Lorebook和记忆管理器可以结合使用，创建更加丰富、连贯的小说内容：

1. 使用Lorebook管理静态世界观设定（角色特征、世界背景等）
2. 使用记忆管理器跟踪动态情节发展（事件进展、角色互动等）
3. 在生成新章节时，系统会自动结合两者的信息，确保内容的一致性和连贯性

## 数据导出和备份

两个系统都支持数据导出和导入功能：

1. 在Lorebook面板中，点击"导出数据"可以导出所有Lorebook条目
2. 在记忆管理器设置面板中，点击"导出记忆"可以导出所有记忆数据
3. 导出的数据为JSON格式，可以保存为备份或在不同项目间共享

## 故障排除

如果遇到问题，请尝试以下解决方法：

1. **Lorebook条目不触发**：检查关键词是否正确，尝试增加更多关键词或提高优先级
2. **记忆系统不工作**：检查记忆管理器设置，确保已启用相关功能
3. **生成内容不连贯**：尝试增加记忆的最大章节数，或减少压缩比例
4. **浏览器性能问题**：如果遇到性能问题，尝试减少记忆的最大章节数或增加压缩比例

## 技术说明

这些功能使用JavaScript实现，数据存储在浏览器的localStorage中，同时也会在服务器上保存备份。系统会自动拦截和增强原有的提示词生成过程，无需手动干预。
